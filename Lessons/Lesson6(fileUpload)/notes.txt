Папка service використовується для файлів які безпосередньо звертаються в  базу.

REST - це архітектурний стиль взаємодії елеменетів мережі між собою. Rest приніс методи POST GET PUT DELETE.
CRUD - CREATE READ UPDATE DELETE!
REST говорить нам що всі урли йдуть в множині, передача даних йде в JSON, маєм структурованість наших запитів:
POST - створити CREATE
GET - взяти READ
PUT - оновити UPDATE
DELETE - видалити DELETE
CRUD!!!

JWT - JSON WebToken  використовується для секюрності.
Token - це захешовані дані.
Токени мають час життя.

папка helpers - для роботи з токенами і різна інша допоміжна штука.

є access token і refresh token
access token - з яким ми ходимо постійно по нашому сервісі
refresh token - який ми використовуєм для формування нової пари токенів коли старий токен вмирає
